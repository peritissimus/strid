01 — The Problem

Your documents are leaking.

Bank statements, medical records, tax returns — they're full of personal data. Sharing them means exposing names, account numbers, Aadhaar numbers, PAN cards, and more. Most redaction tools send your data to the cloud, defeating the purpose entirely.

Strid processes everything on your device. Your documents never leave your machine. Not a single byte goes to any server.
  • 20+ entity types — names, emails, phones, bank accounts, IFSC, PAN, Aadhaar, UPI IDs, and more
  • Context-aware — smart enough to redact dates of birth but preserve transaction dates
  • Two platforms — Python CLI for desktop, Swift iOS app for mobile
  • First of its kind — the only open source iOS app for on-device document redaction

03 — CLI

Three commands. That's it.

The Python CLI is built with typer and powered by Microsoft Presidio + spaCy. Install with uv and you're ready.

detect — find PII without changing anything
# Show detected PII in a table
$ strid detect bank_statement.txt

# With surrounding context
$ strid detect bank_statement.txt -c

# Output as JSON for scripting
$ strid detect bank_statement.txt --json

# Lower threshold to catch more
$ strid detect bank_statement.txt -t 0.3
redact — replace PII with type placeholders
# Redact to stdout
$ strid redact bank_statement.txt

# Redact to file
$ strid redact bank_statement.txt -o clean.txt

# Only redact specific entity types
$ strid redact bank_statement.txt -e PERSON -e IN_PAN -e IN_AADHAAR

# Preview without redacting
$ strid redact bank_statement.txt --dry-run
entities — list all supported PII types
$ strid entities

05 — iOS

First open source iOS redaction app.

StridKit is a standalone Swift package powered by Apple's NaturalLanguage framework and NSDataDetector. All 20+ recognizers ported to Swift with context-aware scoring. No external dependencies. No network calls. Runs on-device.

1

Import

Open a document from Files, paste text, or scan a paper document with the built-in camera.

2

Detect

Strid scans for PII and shows a summary: 43 entities found — 12 txn refs, 5 UPI IDs, 4 locations, 2 IFSC codes, 1 PAN…

3

Review

Every finding listed with type, matched text, confidence score, and color-coded badges. Verify before redacting.

4

Redact & Share

One tap to redact. Copy to clipboard, share via AirDrop, or save to Files. The original stays untouched.

StridKit — Swift API
import StridKit

let engine = StridEngine(threshold: 0.5)

// Detect PII
let entities = engine.detect(in: documentText)
// → [PIIEntity(type: .inPAN, text: "BVMPS4521K", score: 1.0), ...]

// Redact with placeholders
let clean = engine.redact(documentText, style: .placeholder)
// → "PAN: <IN_PAN> VERIFIED"

// Or with asterisks
let masked = engine.redact(documentText, style: .asterisks)
// → "PAN: ********** VERIFIED"

07 — FAQ

Common questions.

General

Does any data leave my device?

No. Strid runs entirely on your device — both the Python CLI and the iOS app. There are zero network calls. No telemetry. No analytics. Your documents stay on your machine.

Why not just use find-and-replace?

Find-and-replace requires you to know exactly what to look for. Strid uses NLP to detect names and locations you might miss, regex with checksum validation for structured data like PAN and credit cards, and context awareness to distinguish a date of birth from a transaction date.

What languages are supported?

The CLI uses spaCy's en_core_web_lg model, so English NER works best. Regex-based recognizers (PAN, Aadhaar, IFSC, etc.) are language-independent. The iOS app uses Apple's NaturalLanguage framework which supports multiple languages for NER.

Will transaction dates get redacted?

No. Strid uses context-aware date detection. Only dates near keywords like "birth", "dob", or "born" are flagged. Regular transaction dates (01/09/25, 02/09/25) are preserved.

Technical

What powers the detection engine?

Python CLI: Microsoft Presidio (analyzer + anonymizer) with spaCy en_core_web_lg for NER, plus custom regex recognizers for Indian banking PII.
iOS: Apple NaturalLanguage framework for NER, NSDataDetector for emails/phones/URLs, and ported regex recognizers with context scoring.

Can I add custom entity types?

Yes. The Python engine accepts custom Presidio recognizers. Create a PatternRecognizer with your regex and context words, then register it with the analyzer. On iOS, implement the Recognizer protocol or create a new RegexRecognizer instance.

How accurate is it?

Structured data (PAN, IFSC, email, credit cards) is near-perfect thanks to pattern + checksum validation. NER-based detection (names, locations) typically achieves F1 ~0.85. You can tune the --threshold flag to trade precision for recall.

What are the system requirements?

CLI: Python 3.12+, runs on any machine (CPU only, no GPU needed). About 500MB disk for spaCy model.
iOS: iOS 17+. No downloads needed — Apple's NLP models ship with the OS.
